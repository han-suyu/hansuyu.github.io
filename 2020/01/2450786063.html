Github+PicGo图床使用

一、七牛云

最开始尝试的是七牛云，毕竟比较有名，上传效果很好，外链也没有问题。

但部署完成后打开博客发现图片加载失败，F12检查后发现“no such domain”的问题。

查阅官方文档后发现了“测试域名”和“自定义域名”的坑。即测试域名只允许用30天，到期自动回收，而后需要用一个已经备案的域名来绑定此域名。。。

虽然没有解决我当前的问题，但当前的发现让我对七牛云好感度-1000

见：https://developer.qiniu.com/kodo/kb/5443/no-to-domain-related-issues

纠结一段时间之后我只好放弃七牛云，寻找其他的图床服务。

知乎上有一个总结，感觉写的很好。

文末有总结：

如果打算长期稳定使用请优先选择又拍云或者七牛云，如果是存储并不重要的图片可以使用免费不限大小的SM.MS图床。

见：https://zhuanlan.zhihu.com/p/35270383

令见：https://www.bilibili.com/video/av31103221/

二、SM.MS

SM.MS，无需注册、实名认证（七牛云需要），上传非常便捷，支持批量上传，但最多10个，上传后即刻给出该图片外链。**同样支持给出markdown链接。**确实很不错。

但过于简洁，不用登录注册，不用绑定任何东西。反倒让我有点担心它的稳定性，万一图片丢了，找都没处去找。。。emmm，就当作一个备用的吧。

再继续找，找到了这个PicGo，一个图床管理工具，顺着PicGo我找到了Github作为图床这一方案。Github，对啊，我直接就确认了这种方案。虽然有人说Github图片国内访问会慢一些，但是它肯定能保证数据的稳定性啊。并且我的博客就是基于github的，这也算是落叶归根吧！好了，就这个了！

三、Github+PicGo

1、新建仓库，设为public

 

  

  

2、获取授权token

Github首页 -> Setting -> Developer settings -> Personal access tokens -> Generate new token -> 确认

按此步骤即可生成token如下，该序列只显示一次，忘记只能重新生成。复制下来保存

 

 

 

3、下载PicGo客户端

官网（特别慢）：https://github.com/Molunerfinn/PicGo/releases

版本2.2.1（蓝奏云下载）：https://www.lanzous.com/i8rh45g

 

 

 

4、安装后配置

  • 1： [github用户名]/[仓库名]

  • 2：master

  • 3：自己的token序列

  • 4：/img （代表图片将上传到仓库下的img文件夹）

  • 5：https://raw.githubusercontent.com/[github用户名]/[仓库名]/master

    设为默认图床后点击确定

 

 

 

5、在上传区上传图片

拖拽上传，也可以剪切板上传，当使用快捷键截屏后再使用Ctrl+Shift+P 自动上传

 

 

 

6、上传成功后在相册查看，同时，github中也会有相应的相册
